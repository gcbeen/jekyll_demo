基本用法

  • git status:查看工作目录的状态
  • git add files | .:把指定的文件存入缓存区域
  • git commit -m 'message': 把缓存区域生成快照并提交
  • git reset -- files: 用来撤销最后一次git add files，你也可以用git reset撤销所有暂存区域文件
  • git checkout -- files 把文件从暂存区域复制到工作目录，用来丢弃本地修改。

git reset -p, git checkout -p, or git add -p进入交互模式。

跳过暂存区域直接从仓库取出文件或者直接提交代码。

  • git commit -am 'message'： 相当于运行 git add . 和 git commit -m 'message'。
  • git commit files -m 'message': 相当于 git add files 和 git commit -m 'message'。
  • git checkout HEAD -- files 回滚到复制最后一次提交。

命令详解

  • Diff查看两次提交之间的变动
    • git diff:缓存区域和工作目录
    • git diff HEAD:current branch和工作目录
    • git diff --cached:缓存区域和current branch
    • git diff other_branch:other branch和工作目录
  • Commit
    • git commit:提交当前分支
    • git commit --amend:覆盖上一次提交
  • Checkout
    • git checkout branch:切换分支
    • git checkout HEAD -- files:回滚到最后一次提交
    • git checkout remotes/origin/gh-pages: 接换到一个匿名分支,git commit提交匿名分支，git checkout -b new_branch 保存匿名分支。
  • Reset
    • git reset:分支指向不变，索引回到最后一次。
    • git reset HEAD~3:分支指向HEAD~3,索引改变(--soft:都不变)，工作目录不变(--hard:工作目录也改变)。
    • git reset -- file：相当于git checkout -- files(reset 索引不变)
  • Merge
    • git merge other:合并当前和other分支，
  • Cherry Pick
  • Rebase
    • git rebase master
    • git rebase --onto master 节点名:
